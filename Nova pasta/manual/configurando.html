Os arquivos do FolhaLivre referentes aos binários, base de dados e código-fonte estão armazenados na área de download do Projeto FolhaLivre no SourceForge.net. Estes arquivos possuem um sufixo no formato x.y, onde x.y é o número da versão estável. De preferência, sempre escolha a versão mais recente destes arquivos.

Os outros arquivos aqui mencionados estão armazenados na área de arquivos do grupo de discussão FolhaLivre em suas respectivas pastas. Lembramos que para ter acesso a área de arquivos do grupo é obrigatório que você esteja logado no Yahoo!Grupos. Esclarecemos que essa obrigatoriedade não faz parte da política do Grupo FolhaLivre e sim do Yahoo!Grupos.

Instalação do sistema gerenciador de banco de dados Interbase ou Firebird

  • Faça o download do Firebird 1.5 (o FolhaLivre ainda não é compatível com a versão 2) em http://www.firebirdsql.org/ para a plataforma win32. Opcionalmente o Firebird 1.5 pode ser baixado da área de arquivos do grupo FolhaLivre (pasta SGDB);
  • Execute o programa de instalação do Firebird baixado e instale-o completamente, escolhendo a opção "Full installation of Super Server e Development Tools" para o servidor (computador onde ficará a base de dados);
  • Para os computadores que serão os clientes, execute o programa de instalação do Firebird e escolha a opção "Minimun client install - no server, no tools";
  • O Firebird precisa está em execução na máquina servidor para que o FolhaLivre funcione adequadamente.

Instalação do binário e da base de dados

  • Para o máquina servidor

    1. Faça o download do arquivo flivre-binario-x.y.zip. Descompacte-o (extrair) o arquivo baixado para pasta c:\flivre\ ou para outra pasta de sua preferência. Esta pasta conterá os sequintes arquivos:

      flivre.exe - Binário (executável);
splashscreen - Imagem de apresentação em formato bitmap;
licence-gpl.txt - Contrato da Licença GNU-GPL;
flivre.ini - Configuração;
update.txt - Histórico das modificações;
alguns arquivos jpeg - Papéis de parede.

    2. Faça o download do arquivo flivre-data-x.y.zip. Descompacte-o (extrair) para a mesma pasta onde encontra-se o binário. Este arquivo contém o arquivo flivre.gdb.

    3. O arquivo flivre.ini deve está localizado na mesma pasta do binário flivre.exe e possuir a chave DATABASE na seção [DATA] que informa o caminho da base de dados. Se o arquivo flivre.ini não existe ou a chave DATABASE estiver em branco o sistema procurará a base de dados na mesma pasta onde o mesmo está localizado.

  • Para a máquina cliente

    1. Repita o passo 1 da instalação para a máquina servidor;

    2. Altere no arquivo flivre.ini a entrada DATABASE para informar a localização da base de dados (que deverá estar na máquina "servidor").

      Abaixo um exemplo do conteúdo de flivre.ini:

      [DATA]
SERVER=servidor
CONECTOR=:
DATABASE=c:\flivre\flivre.fdb

    ou ainda,

    [DATA]
SERVER=10.41.0.127
CONECTOR=:
DATABASE=c:\flivre\flivre.fdb

  • Para a atualização servidor/cliente

    1. Faça o download do arquivo flivre-binario-x.y.zip. Escolha Sempre a versão mais recente.
    2. Descompacte (extrair) o arquivo baixado para pasta onde está instalado o FolhaLivre, substituindo o arquivo flivre.exe tanto na máquina "servidor" quanto na(s) máquina(s) "cliente" (se for o caso).
    3. Execute o FolhaLivre da máquina servidor e escolha a opção <Atualização da Base de Dados> do menu <Ferramentas> e confirme a operação.
    4. Importante: O arquivo flivre-data-x.y.zip deve ser baixado com cautela, pois sua descompactação para a pasta onde estar a base de dados provocará a perda da base de dados antiga.

Instalação dos códigos-fontes

Esta seção é exclusiva para desenvolvedores e exige um relativo conhecimento do programa Borland Delphi/Kylix. Se você não é desenvolvedor ou não tem interesse em programação, por favor desconsidere esta parte.

  • Descompacte o arquivo flivre-source.zip na pasta c:\flivre\ ou em outra pasta mantendo a hierarquia das subpastas. O nome da pasta onde você descompactou o arquivo flivre-source.zip será chamada de ~ por questão de praticidade. Depois da descompactação são criadas as sequintes pastas::

    ~\source\         - arquivos de fontes exclusivos do FolhaLivre;
~\source\kernel\  - núcleo básico para vários projetos;
~\aklib\          - arquivos da biblioteca AKLib.

  • No Borland Delphi/Kylix acesse a opção Environment Options... no menu Tools. Na guia (aba) Library esolha o botão ... em Library Path e adicione a pasta ~\source\kernel\ na lista de pastas apresentadas. Isto é necessário para que o Delphi/Kylix encontre as respectivas units (arquivos/unidades) durante o desenvolvimento e compilação, pois as mesmas não são adicionadas explicitamente no projeto flivre.dpr.

Bibliotecas

Esta seção é exclusiva para desenvolvedores e exige um relativo conhecimento do programa Borland Delphi/Kylix. Se você não é desenvolvedor ou não tem interesse em programação, por favor desconsidere esta parte.

O FolhaLivre utiliza algumas bibliotecas de componentes para auxiliar no desenvolvimento. Estas bibliotecas não fazem parte do Projeto FolhaLivre, com exceção da AKLib, sendo de seus respectivos autores a responsabilidade de seus aprimoramentos.

Seus códigos fontes estão disponíveis e suas licenças permitem sua inclusão em um software livre/aberto.

As bibliotecas utilizadas pelo FolhaLivre são as sequintes:

  • AKLib [http://br.groups.yahoo.com/group/folha_livre/];
  • SynEdit [http://synedit.sourceforge.net/];
  • MDIWallPaper [http://home.t-online.de/home/Robert.Rossmair/].

Instalação das Bibliotecas

  • Faça o download das bibliotecas, com exceção da AKLib, que vem junto ao código-fonte do FolhaLivre e encontra-se na pasta ~\aklib\;

  • Descompacte cada biblioteca em uma pasta diferente mantendo a hierarquia das subpastas;

  • No Borland Delphi/Kylix acesse a opção Environment Options... no menu Tools. Na guia Library clique no botão [...] em Library Path. Na lista apresentada inclua a pasta onde estão as units *.pas da biblioteca. Isso permite que o Borland Delphi/Kylix encontre todas as units necessárias à compilação.

    Por exemplo, se você descompactou a biblioteca SynEdit.zip na pasta de instalação do Borland Delphi/Kylix, informe na lista a seguinte entrada: $(DELPHI)\SynEdit\Source\.

  • No menu principal do Borland Delphi/Kylix File->Open, na caixa de diálogo Open altere a opção Arquivos do Tipo para Delphi package (*.dpk), mude para a pasta onde está o pacote dpk (package) da biblioteca a ser instalada correspondente à versão do Delphi/Kylix, pressione o botão [Abrir].

  • Uma caixa de diálogo Package será apresentada, pressione o botão [Install] e a partir deste ponto o Borland Delphi/Kylix compilará as units e tentará instalar no IDE os respectivos componentes. Se nenhum erro for encontrado, uma janela de confirmação será mostrada. Se a instalação foi bem sucedida feche a caixa de diálogo "Package".

  • O projeto flivre.dpr está configurado para gerar o executável na pasta ~.
